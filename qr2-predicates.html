Topic 1 · Unary Predicates — The Fundamental Break from Propositional Logic
TOPIC 1
Unary Predicates — What Makes P(x) Different from p
Phase 1 — Translation (English Only)

In propositional logic, p is a complete, self-contained sentence: "The defendant is liable." It is either true or false, full stop. There is nothing to plug in, nothing to evaluate — it's just a fact about the world, fixed.

A predicate is different in one crucial way: it is a template, not a sentence. "x is liable" is not yet a statement — it has a blank in it (the x). It only becomes true or false once you plug something specific into that blank. Until then, it's a question waiting to be asked of each possible subject.

Phase 1 Test — English Recognition

Before writing a single symbol, ask: "Is this complete, or does it have a blank?"

"The defendant was negligent." → Complete, no blank → propositional (p, q, r).

"x was negligent." → Has a blank (who is x?) → predicate, needs a subject to be evaluated.

"x employed y." → Has two blanks (who employs whom?) → predicate with two variables.

If you can't determine whether the sentence is true or false without knowing "who is x," you have a predicate. This test works even when x is disguised as "the person," "any contractor," or "someone."

The Disguise Pattern

PU and LSAT problems almost never present predicates with x explicitly labelled. Instead: "A contractor who misses a deadline is liable for penalties." This is the predicate L(c) = "contractor c is liable" applied to all contractors who miss deadlines — the x is hidden inside "a contractor" and "who misses a deadline." Recognising predicates through their natural-language disguises is the entire skill of Phase 1.

Phase 2 — Mechanics & Notation
Unary predicate: P(x) — one variable, one property
Read as "x has property P" or "P holds for x"

Domain: the set of things x is allowed to be
P(x) = "x is a licensed lawyer" over the domain {all persons} gives a different truth set than over {all lawyers in Pakistan}

Evaluating a predicate at a specific individual:
Substitute the individual for x, then determine true or false
P(a) = "a is licensed" — true if a is licensed, false if not

Compound predicates (connectives apply to predicates too):
P(x) ∧ Q(x) — x has both properties P and Q
P(x) ∨ Q(x) — x has at least one of P or Q
¬P(x) — x does not have property P
P(x) → Q(x) — if x has P, then x has Q (still not a statement until x is specified)

Key distinction from propositional logic:
p ∧ q — two specific propositions conjoined → always has a fixed truth value
P(x) ∧ Q(x) — still a template → truth value depends on what x is
Phase 3 — Full Pipeline Worked Example
English → Predicate → Evaluate → English Check

Given: "A lawyer who holds a valid practising certificate is authorised to appear in the High Court."

Step 1 — Phase 1 check: "a lawyer" introduces a blank subject → this is a predicate statement, not a propositional one. It makes a claim about each lawyer individually, not about a single named person.

Step 2 — Define predicates and domain:

Domain: all lawyers registered in Pakistan
C(x) = "x holds a valid practising certificate"
A(x) = "x is authorised to appear in the High Court"
Formalisation: C(x) → A(x)   [for each x in the domain]

Step 3 — Evaluate at a specific individual: Lawyer Ali holds a valid certificate. So C(Ali) = T. By the conditional: C(Ali) → A(Ali), since C(Ali) = T, we conclude A(Ali) = T. Ali is authorised.

Step 4 — Check a second individual: Lawyer Raza has an expired certificate. C(Raza) = F. The conditional C(Raza) → A(Raza) has a false antecedent — it is vacuously true but tells us nothing about A(Raza). We cannot conclude Raza is authorised, but we also cannot conclude he is not authorised from this rule alone. A false antecedent leaves the consequent completely undetermined.

Step 5 — English check: reading C(x) → A(x) back into English: "if x holds a valid certificate, then x is authorised" — this matches the original sentence, confirming the formalisation is correct in both directions.

T2 · AClassify each as propositional (p) or predicate (P(x)): (i) "The contract is void." (ii) "A contract signed under duress is voidable." (iii) "If x is a minor, x cannot form a binding contract." (iv) "This verdict was unanimous."

(i) Propositional — "the contract" is a specific named thing; the sentence is complete and has a fixed truth value.

(ii) Predicate — "a contract" introduces a blank; the sentence makes a claim about each contract individually: V(x) → D(x) where V(x)="x was signed under duress" and D(x)="x is voidable."

(iii) Predicate — explicitly has x as a placeholder; M(x) → ¬C(x) where M(x)="x is a minor" and C(x)="x can form a binding contract."

(iv) Propositional — "this verdict" refers to a specific verdict; the sentence is complete with a fixed truth value.

T2 · CWrite the negation of P(x) ∧ Q(x) using De Morgan's law. Then interpret: P(x) = "x filed the required disclosure" and Q(x) = "x filed within the deadline." What does ¬[P(x) ∧ Q(x)] mean in plain English, and is it the same as ¬P(x) ∧ ¬Q(x)?

By De Morgan: ¬[P(x) ∧ Q(x)] = ¬P(x) ∨ ¬Q(x)

Plain English: "x did NOT (file the required disclosure AND file within the deadline)" = "x either failed to file the disclosure OR filed after the deadline (or both)." This is the correct negation.

Is it the same as ¬P(x) ∧ ¬Q(x)? No. ¬P(x) ∧ ¬Q(x) means "x did not file the disclosure AND did not file within the deadline" — a person who filed late (¬Q but P is true) satisfies ¬[P∧Q] but not ¬P∧¬Q. The difference matters legally: a filing that arrived but was late violates the conjunction but is not the same as no filing at all.

T2 · DDomain: all documents submitted to a tribunal. L(x) = "x is in the required language." A(x) = "x is authenticated." R(x) = "x will be received." Rule: R(x) ↔ [L(x) ∧ A(x)]. Evaluate R for: Document D1 (correct language, authenticated), Document D2 (correct language, not authenticated), Document D3 (wrong language, authenticated).

The rule is a biconditional: R(x) if and only if both L(x) and A(x) hold.

D1: L(D1)=T, A(D1)=T → L∧A=T → R(D1)=T. Document received.

D2: L(D2)=T, A(D2)=F → L∧A=F → R(D2)=F. Not received — missing authentication.

D3: L(D3)=F, A(D3)=T → L∧A=F → R(D3)=F. Not received — wrong language.

The biconditional is stricter than a one-way conditional: both conditions are simultaneously necessary and jointly sufficient. A document satisfying only one is not received, regardless of which one it satisfies.

T3 · AA statutory clause reads: "A person is exempt from the surcharge if, and only if, they are a registered charity AND their annual income does not exceed the prescribed threshold." (a) Formalise this as a predicate logic statement. (b) The revenue authority argues that Organisation X is not exempt because it exceeded the income threshold last year, even though it is a registered charity. Is this argument structurally valid? (c) Organisation X argues that even if it exceeded the threshold, it should be exempt because it is a registered charity and charities serve the public good. Evaluate this counter-argument logically and legally.

(a) Domain: all organisations. C(x)="x is a registered charity." I(x)="x's annual income does not exceed the prescribed threshold." E(x)="x is exempt from the surcharge." Formalisation: E(x) ↔ [C(x) ∧ I(x)]

(b) The revenue authority's argument is structurally valid. If Organisation X exceeded the income threshold, then I(X)=F. By the biconditional, E(X) ↔ [C(X) ∧ I(X)] — since I(X)=F, the conjunction C(X)∧I(X)=F, and therefore E(X)=F. The exemption requires both conditions simultaneously; failing one is sufficient to deny the exemption regardless of whether the other holds. The argument correctly applies the biconditional.

(c) Organisation X's counter-argument is logically invalid against the statute as written, and legally weak as a primary argument though potentially viable as a policy argument. Logically: the statute's exemption criterion is E(x) ↔ [C(x) ∧ I(x)]. The fact that charities serve the public good is a reason the legislature may have included charities in the exemption, but it is not an additional exemption criterion in the text. Adding "serves the public good" as a standalone exemption criterion requires amending the statute, not interpreting it. Legally: if Organisation X can argue the income threshold was exceeded due to one-time anomalous income that doesn't reflect ongoing financial capacity, it might seek a purposive interpretation — but that is a legal argument about legislative intent, not a logical argument that the biconditional doesn't apply. On the plain text, the revenue authority's position is correct.

T3 · BA regulation states: "A food business operator is required to maintain temperature logs if they store perishable goods, OR if they employ more than five food handlers." Let S(x) = "x stores perishable goods," H(x) = "x employs more than five food handlers," L(x) = "x is required to maintain temperature logs." (a) Formalise the regulation. (b) A business operator stores no perishable goods and employs three food handlers — are they required to maintain logs? (c) The regulator issues an enforcement notice to a business that stores no perishable goods but employs eight food handlers. The business argues "we don't need logs because we don't store perishables." Evaluate the argument. (d) Propose a stricter version of the regulation that would require logs only when both conditions hold, and explain the practical consequence of the change.

(a) [S(x) ∨ H(x)] → L(x)

(b) S(x)=F, H(x)=F → S∨H=F → antecedent false → conditional vacuously true, but L(x) is not forced to be true. The business is NOT required to maintain logs.

(c) S(x)=F, H(x)=T → S∨H=T (disjunction requires only one to hold) → antecedent true → L(x)=T. The business IS required to maintain logs. The business's argument incorrectly applies the rule: it treats the disjunction as if both conditions must hold (a conjunction), when in fact either condition alone suffices. This is a formal fallacy — the business denies the consequent by eliminating only one disjunct of the antecedent, but the other disjunct (H) remains true.

(d) Stricter version: [S(x) ∧ H(x)] → L(x). Under this rule, logs are required only when a business both stores perishables AND employs more than five handlers. Practical consequence: the business in (c) — eight food handlers, no perishables — would no longer be required to maintain temperature logs under the stricter rule. The regulatory burden is reduced, but a food safety gap opens: large food-handler operations without perishable storage may still present temperature-related risks (e.g., prepared hot foods, equipment cleaning requirements) that fall outside the new, narrower trigger.

Bar & LSAT Lens
NY Bar · Statutory Drafting & LSAT Logic Games

Multi-element statutory triggers are predicate logic in legislative form. The exemption/liability determination for any given party (the x) requires evaluating each predicate in the trigger condition, applying the connective (∧, ∨, ↔), and reading off the result. A lawyer who can do this systematically never makes the "I don't qualify for X because I lack element A" error when element B alone might suffice — and never concedes "my client is subject to rule R" without checking every element of R's trigger condition individually.

LSAT Logic Games are almost entirely predicate logic in disguise: "If a lawyer is admitted to the state bar AND has not been disciplined in the past three years, they may appear pro hac vice." Every game rule of this form is a conditional predicate. The formal evaluation procedure — plug in each player, check each predicate, apply the connective, read the result — is the exact mechanical skill Logic Games reward when applied quickly and without error.

Topic 2 · Binary Predicates — Relationships Between Two Variables
TOPIC 2
Binary Predicates — R(x, y) and Relationships Between Subjects
Phase 1 — Translation (English Only)

A unary predicate has one blank — one subject that needs filling. A binary predicate has two blanks — it describes a relationship between two things. "x employs y," "x is the supervisor of y," "x filed a claim against y" — all of these need two subjects to become a complete true/false statement.

Phase 1 Test — Spotting Two Blanks

Ask: how many things do I need to name before this sentence has a truth value?

"x is negligent." → one thing (x) → unary P(x).

"x supervises y." → two things (x and y) → binary R(x,y).

"x transferred funds to y on behalf of z." → three things → ternary predicate (rare in QR-II but good to recognise).

The order of the variables matters in binary predicates: R(x,y) = "x employs y" is NOT the same as R(y,x) = "y employs x." Always state which is x and which is y before evaluating.

The Order Trap

In English, "Ali supervises Sara" and "Sara supervises Ali" are clearly different. In predicate notation, S(Ali, Sara) ≠ S(Sara, Ali). But when a word problem uses passive voice — "Sara is supervised by Ali" — students often reverse the order in the predicate without noticing. Always rewrite passive voice as active voice before formalising: "Sara is supervised by Ali" → "Ali supervises Sara" → S(Ali, Sara).

Phase 2 — Mechanics & Notation
Binary predicate: R(x, y) — two variables, a relationship between them
Read as "x stands in relation R to y" or "R holds between x and y"

Domain: now two domains — one for x, one for y (often the same set, sometimes different)
e.g. E(x, y) = "x employs y" where x ∈ {companies} and y ∈ {persons}

Evaluating at specific individuals:
E(TechCo, Ayesha) = T if TechCo employs Ayesha, F otherwise
E(Ayesha, TechCo) = ? — a different statement (does Ayesha employ TechCo?), usually F

Compound binary predicates:
E(x,y) ∧ P(y) — "x employs y AND y is a minor"
R(x,y) → Q(x,y) — "if x supervises y then x is responsible for y's conduct"

Chaining predicates:
E(x,y) ∧ E(y,z) → contract chain: x employs y, y employs z — does x have liability for z?
This is the multi-party contract structure that makes binary predicates legally important
Phase 3 — Full Pipeline Worked Example
English → Binary Predicate → Chain Evaluation → English Check

Given: "An employer is vicariously liable for any harm caused by an employee while acting within the scope of their employment."

Step 1 — Phase 1: two blanks — the employer (x) and the employee (y) → binary predicate structure. A third element, the harm, is implied but treated as background.

Step 2 — Define:

Domain: x ∈ {employers}, y ∈ {persons}
E(x,y) = "x employs y"
S(y) = "y was acting within the scope of employment"
H(y) = "y caused harm"
V(x) = "x is vicariously liable for the harm"

Rule: [E(x,y) ∧ S(y) ∧ H(y)] → V(x)

Step 3 — Evaluate: Company C employs Driver D. D caused an accident while making a delivery (within scope). E(C,D)=T, S(D)=T, H(D)=T → conjunction=T → V(C)=T. Company C is vicariously liable.

Step 4 — What if D was on a personal errand? S(D)=F → conjunction=F → V(C) is not forced to be T by this rule. Vicarious liability does not apply. The employer may still be liable on other grounds, but this specific rule doesn't establish it.

Step 5 — English check: [E(x,y) ∧ S(y) ∧ H(y)] → V(x) reads back as "if x employs y, and y was acting within scope, and y caused harm, then x is vicariously liable" — exactly matching the original rule. Formalisation confirmed.

T3 · AA company's data-sharing policy states: "Personal data may be shared with a third party if the data subject has consented AND the third party is a verified partner." Let: S(x,y) = "x shares data about z with y," C(z) = "z has consented," V(y) = "y is a verified partner," P(x,y,z) = "x may permissibly share data about z with y." (a) Formalise the permission rule. (b) The company shares data about User U with Organisation O (not a verified partner) but claims User U's general privacy policy acceptance counts as consent. Is this permissible under the rule? (c) The company then shares data about User U with Partner P (verified partner) without seeking U's specific consent, relying on U's general terms acceptance. Is this permissible? (d) Propose a formalisation of a stricter rule that would require both consent AND partnership AND purpose limitation, and name one additional predicate needed.

(a) Rule: P(x,y,z) ↔ [C(z) ∧ V(y)] — data about z may be shared by x with y if and only if z has consented and y is a verified partner.

(b) Organisation O, User U: V(O)=F (O is not a verified partner) → C(U)∧V(O)=F → P(company,O,U)=F. Not permissible, regardless of whether C(U) is true. The missing partnership status alone makes sharing impermissible under the biconditional rule.

(c) Partner P, User U (no specific consent): V(P)=T (verified partner). Whether C(U)=T depends on whether general terms acceptance constitutes "consent" within the rule's meaning. If the rule requires specific, informed consent to sharing (as under GDPR), general terms acceptance does not satisfy C(U) → P(company,P,U)=F. If general acceptance is sufficient, C(U)=T → P(company,P,U)=T. The predicate logic here correctly identifies that the legal dispute is entirely about the definition of C(z) — a definitional dispute, not a logical one.

(d) Stricter rule: add L(x,y,z) = "the purpose of sharing z's data with y matches the purpose for which z consented." New rule: P(x,y,z) ↔ [C(z) ∧ V(y) ∧ L(x,y,z)]. Now all three conditions must hold simultaneously. A company that shares data for a purpose beyond what the user consented to (e.g., using medical data for marketing when consent was only for treatment purposes) fails L even if C and V hold.

T3 · BA fiduciary duty rule states: "Person x owes a fiduciary duty to person y if x manages assets on y's behalf AND y places trust and confidence in x." Let M(x,y)="x manages assets on y's behalf," T(y,x)="y places trust and confidence in x," F(x,y)="x owes fiduciary duty to y." (a) Formalise. (b) Investment Manager Ali manages a pension fund on behalf of Beneficiary Bano, who trusts Ali with investment decisions. Does Ali owe a fiduciary duty to Bano? (c) Lawyer Zara advises Client Khalid. Khalid trusts Zara's legal judgment. However, Zara does not manage any of Khalid's assets. Does Zara owe a fiduciary duty to Khalid under this rule? (d) This rule is widely criticised as too narrow for capturing the full scope of fiduciary relationships in law. Using predicate logic, propose an alternative rule using a disjunction that captures both asset management and non-asset professional trust relationships.

(a) [M(x,y) ∧ T(y,x)] → F(x,y)

(b) Ali and Bano: M(Ali,Bano)=T, T(Bano,Ali)=T → conjunction=T → F(Ali,Bano)=T. Ali owes a fiduciary duty to Bano.

(c) Zara and Khalid: M(Zara,Khalid)=F (Zara does not manage assets) → M∧T=F → F(Zara,Khalid) not established by this rule. Under this specific formulation, Zara does not owe a fiduciary duty to Khalid — the asset management predicate is not satisfied.

(d) Proposed alternative: add P(x,y)="x exercises professional power or discretion over y's interests." New rule: [M(x,y) ∧ T(y,x)] ∨ [P(x,y) ∧ T(y,x)] → F(x,y), which simplifies to [M(x,y) ∨ P(x,y)] ∧ T(y,x) → F(x,y). Zara now owes a fiduciary duty. This disjunctive formulation captures the broader legal understanding of fiduciary relationships that extends beyond asset management to all relationships of professional power and trust.

Bar & LSAT Lens
NY Bar · Vicarious Liability & LSAT Logic Games

The binary predicate chain — E(x,y) ∧ E(y,z) creating obligations between x and z — is the formal structure of every respondeat superior, agency, and subcontractor liability analysis. The NY Bar's professional responsibility questions routinely test this structure: when does a supervising attorney's P(supervisor, associate) relationship create liability for the associate's actions? The answer is always a conjunction of predicates evaluated against specific facts.

LSAT Logic Games with "groups and members" or "assignment" setups are binary predicate evaluations: assigning person x to project y (A(x,y)=T), subject to constraints of the form "if A(x,y)=T then A(z,y) must also be T." Recognising these as R(x,y) structures and tracking the truth values of each pair systematically is the most reliable strategy for these game types — faster and less error-prone than intuitive methods.

Topic 3 · Predicates Combined with Quantifiers — The Full Structure
TOPIC 3
Predicates with Quantifiers — ∀x P(x) and ∃x P(x) in Full Context
Phase 1 — Translation (English Only)

Adding a quantifier to a predicate is what turns a template into a complete statement. ∀x P(x) says "P holds for every x in the domain." ∃x P(x) says "P holds for at least one x in the domain." Both are now full propositions with definite truth values — not templates anymore.

Phase 1 Test — Which Quantifier Is Hidden Here?

Two questions to ask of any English sentence with a predicate:

Q1: Does this claim need to hold for EVERY member of the group, or only for AT LEAST ONE?

"All lawyers must disclose conflicts." → Every lawyer → ∀

"Some witness statements were inconsistent." → At least one → ∃

"No contractor is exempt." → Every contractor is NOT exempt → ∀x ¬P(x)

"Not all claims were filed on time." → At least one was NOT on time → ∃x ¬P(x)

Q2: What is the domain? The quantifier's scope is always limited to a stated or implied domain. "Every person" and "every registered contractor" are different ∀ statements even if the property P is identical.

The "No" and "None" Trap

"No contractor is licensed" looks like it might be an ∃ (existential negative), but it's actually a universal: ∀x ¬L(x) — "for every contractor x, x is not licensed." The word "no" in English almost always signals ∀ with negation, not ∃. Similarly, "none of the evidence supports this claim" = ∀x ¬S(x), not ∃x ¬S(x).

Phase 2 — Mechanics & Notation
Universal quantifier: ∀x P(x) — "for all x in the domain, P(x) holds"
Proves true: must verify P holds for EVERY element
Proves false: need only ONE counterexample

Existential quantifier: ∃x P(x) — "there exists at least one x in the domain for which P(x) holds"
Proves true: exhibit ONE example where P holds
Proves false: must verify P fails for EVERY element

Quantified conditionals (the most common legal form):
∀x [P(x) → Q(x)] — "for all x, if x has P then x has Q"
NOT the same as: ∀x P(x) → ∀x Q(x) (different logical structure)

Quantified binary predicates:
∀x ∃y R(x,y) — "every x is related to some y" (different y for each x)
∃y ∀x R(x,y) — "some single y is related to every x" (same y for all x)
These two are NOT equivalent — order of quantifiers matters enormously

English signal words:
∀: all, every, any, each, no (with ¬), none (with ¬), whenever, whoever
∃: some, at least one, there exists, there is a, certain, several
Phase 3 — Full Pipeline Worked Example
Full Pipeline — Legal Sentence → Quantified Predicate → Evaluate → English Check

Given: "Every registered company that fails to file its annual return is subject to a penalty, and at least one registered company failed to file last year."

Step 1 — Phase 1: Two separate claims. First: a claim about EVERY registered company with a condition → ∀ with a conditional predicate. Second: a claim about AT LEAST ONE registered company → ∃.

Step 2 — Define:

Domain: all registered companies
F(x) = "x failed to file its annual return"
P(x) = "x is subject to a penalty"

Claim 1: ∀x [F(x) → P(x)]
Claim 2: ∃x F(x)

Step 3 — Derive a conclusion: From ∃x F(x), there is some specific company — call it Company A — where F(A)=T. From ∀x [F(x) → P(x)], apply to Company A: F(A)→P(A). Since F(A)=T, by modus ponens: P(A)=T. Company A is subject to a penalty.

Step 4 — What does ∀x [F(x)→P(x)] say about a company that DID file? For Company B that filed: F(B)=F → F(B)→P(B) is vacuously true. The rule says nothing about whether B is subject to a penalty on other grounds. It only guarantees penalties for failures; it doesn't guarantee freedom from penalties for compliant companies (which might be penalised for other reasons).

Step 5 — English check: ∀x [F(x)→P(x)] reads "for every company x, if x failed to file, then x is subject to a penalty." ∃x F(x) reads "there is at least one company that failed to file." Both match the original. Formalisation confirmed.

T2 · AIdentify the quantifier (∀ or ∃) in each sentence and write the formal predicate statement: (i) "All contracts require consideration." (ii) "Some witnesses recanted their testimony." (iii) "No employee was informed of the policy change." (iv) "Not every appeal was decided within the statutory period."

(i) ∀ — "all" signals universal. ∀x [C(x) → R(x)] where C(x)="x is a contract" and R(x)="x requires consideration."

(ii) ∃ — "some" signals existential. ∃x [W(x) ∧ T(x)] where W(x)="x is a witness" and T(x)="x recanted testimony."

(iii) ∀ with negation — "no" signals universal negation. ∀x [E(x) → ¬I(x)] where E(x)="x is an employee" and I(x)="x was informed of the policy change." Equivalently: ¬∃x [E(x) ∧ I(x)].

(iv) ∃ with negation — "not every" signals that at least one fails. ∃x [A(x) ∧ ¬D(x)] where A(x)="x is an appeal" and D(x)="x was decided within the statutory period."

T2 · BFrom ∀x [L(x) → D(x)] (every licensed professional must disclose conflicts) and the facts: Ali is licensed (L(Ali)=T); Sara is not licensed (L(Sara)=F); Tariq is licensed and has a conflict. (a) Does Ali have a disclosure obligation? (b) Does Sara have a disclosure obligation under this rule? (c) Does Tariq have a disclosure obligation, and does this rule tell us what happens if he doesn't disclose?

(a) Ali: L(Ali)=T → D(Ali)=T by the universal conditional. Ali has a disclosure obligation.

(b) Sara: L(Sara)=F → L(Sara)→D(Sara) is vacuously true, but D(Sara) is not forced to be T. Sara has no disclosure obligation under this rule — but note the rule says nothing about whether Sara might have other obligations under other rules (e.g. as an unlicensed person acting in a professional capacity).

(c) Tariq: L(Tariq)=T → D(Tariq)=T. Yes, Tariq has a disclosure obligation. However, ∀x [L(x)→D(x)] only establishes the obligation — it says nothing about consequences of non-disclosure. The rule tells us Tariq must disclose; a separate rule (penalty, liability, disqualification) would be needed to determine what happens if he doesn't. This file's rule is obligation-creating, not consequence-specifying.

T2 · CFormalise and evaluate: "Every supplier who receives a purchase order must acknowledge receipt within 24 hours. Supplier S received three purchase orders. At least one was not acknowledged within 24 hours." What conclusion can be drawn, and does the rule tell you anything about the unacknowledged order specifically?

Domain: all purchase orders received by suppliers. R(x)="x received a purchase order." A(x)="x acknowledged within 24 hours." Rule: ∀x [R(x)→A(x)]. Fact 1: R(o₁)=T, R(o₂)=T, R(o₃)=T for Supplier S's three orders. Fact 2: ∃x ¬A(x) — at least one order was not acknowledged.

Conclusion: since ∀x [R(x)→A(x)] requires acknowledgment for every received order, and ∃x ¬A(x) shows at least one order was not acknowledged despite being received, Supplier S has violated the rule. The existential fact (at least one unacknowledged) directly contradicts the universal obligation (all must be acknowledged).

What the rule says about the specific unacknowledged order: the rule itself only establishes the obligation. A breach has occurred (the universal was violated), but whether this triggers a penalty, gives the buyer a right to cancel, or constitutes a contract breach requires additional rules. The predicate analysis correctly identifies that a violation occurred — it doesn't specify the consequence.

T2 · DExplain the difference between ∀x ∃y R(x,y) and ∃y ∀x R(x,y). Give a legal example for each, and explain why a lawyer who confuses them makes a material error in a contract analysis context.

∀x ∃y R(x,y): "For every x, there exists some y (possibly different for each x) such that R(x,y) holds." Every x has its own y.

Legal example: "Every contractor has at least one supervisor." Each contractor has a supervisor, but different contractors can have different supervisors — there is no single supervisor for all.

∃y ∀x R(x,y): "There exists a single y such that for every x, R(x,y) holds." One y works for all x simultaneously.

Legal example: "There is a single compliance officer responsible for every contractor." One specific person supervises all contractors — a much stronger, centralised claim.

Why confusing them is a material error: In a contract context, "every party has at least one point of contact" (∀x∃y) means each party may deal with different representatives — no single person is responsible for the whole relationship. "There is a single contract manager for all parties" (∃y∀x) means one person is accountable across the board. If a lawyer drafts a coordination clause using ∀x∃y language thinking it creates a single point of accountability, the clause actually permits different contacts for each party — the opposite of what was intended. The error creates a governance gap with real consequences in multi-party contracts.

Quantifier order · ∀∃ vs ∃∀ structural difference · legal examples for each · material drafting consequence of confusing the two
T3 · AA constitutional provision states: "Every citizen is equal before the law, and no citizen shall be denied equal protection." A government regulation exempts a specific professional class from a licensing requirement that applies to all other citizens. (a) Formalise both the constitutional provision and the regulation using predicate logic. (b) Identify the logical tension between them. (c) The government argues the exemption is permitted because "equality before the law does not mean identical treatment." Evaluate this argument using your formalisations — is there a logical formulation of "equality" that is consistent with the regulation, and what would it require?

(a) Domain: all citizens. C(x)="x is a citizen." E(x)="x is equal before the law." P(x)="x is denied equal protection." L(x)="x is subject to the licensing requirement." Prof(x)="x is in the exempt professional class." Constitutional provision: ∀x [C(x)→E(x)] ∧ ∀x [C(x)→¬P(x)]. Regulation: ∀x [C(x) ∧ ¬Prof(x) → L(x)] ∧ ∀x [C(x) ∧ Prof(x) → ¬L(x)].

(b) Logical tension: ∀x [C(x)→E(x)] asserts every citizen has equal status before the law. The regulation creates a class of citizens (Prof(x)) who are treated differently with respect to L(x) — some citizens face the licensing requirement, others do not. If E(x) means "x is treated identically to all other citizens in all respects," this is a direct contradiction. Under that interpretation, the regulation violates ∀x [C(x)→E(x)].

(c) The government's argument: the argument is logically coherent if "equality" is given a weaker, relational definition. Specifically, if E(x) means "x's treatment is determined by the same rational criteria applied uniformly to their category" rather than "x receives literally identical treatment as all others," then the regulation is consistent — provided that the categorisation into Prof(x) and ¬Prof(x) is itself based on a legitimate, consistently applied criterion (e.g., professional training requirements that make the licensing redundant). Formally, this requires adding a predicate J(x,c)="x's treatment under criterion c is justified by a legitimate distinction" and reformulating E(x) as "x's treatment satisfies J(x,c) for all applicable criteria c." Under this formulation, the regulation is consistent with equality if the professional class distinction satisfies J — i.e., if there is a rational basis for the exemption. This is structurally exactly the rational basis review standard in constitutional law: the logical question of consistency depends entirely on how the predicate E(x) is defined, which is the legal question courts resolve through constitutional interpretation.

T3 · BTwo statutory rules: Rule 1: ∀x [A(x) ∧ B(x) → C(x)]. Rule 2: ∃x [A(x) ∧ ¬C(x)]. (a) Are these rules consistent or contradictory? Show your reasoning. (b) Applying to: A(x)="x is an accredited laboratory," B(x)="x tested the sample," C(x)="x's results are admissible as evidence." The prosecution argues: "Lab X is accredited and tested the sample, so results are admissible." The defence finds evidence that Lab X's results were excluded in a previous case. Does the defence's finding contradict Rule 1, and what must the defence actually prove to exclude the evidence in this case?

(a) The rules are consistent — no contradiction. Rule 1 says every x with both A and B has C. Rule 2 says some x has A without C. These are compatible: the ∃x in Rule 2 could be a laboratory that has accreditation (A) but did NOT test the sample (¬B) — so it satisfies A∧¬C without violating Rule 1, because Rule 1 only guarantees C when BOTH A and B hold. If Rule 2's x has A and ¬B, the antecedent of Rule 1 is false for that x, and Rule 1 is vacuously satisfied. The two rules would only be contradictory if Rule 2's x had both A and B and ¬C — that would directly contradict Rule 1.

(b) The prosecution's argument: A(LabX)=T (accredited), B(LabX)=T (tested the sample) → by Rule 1, C(LabX)=T. The argument is structurally valid under Rule 1.

The defence finding — Lab X's results were excluded previously — does not contradict Rule 1. Past exclusion means C(LabX)=F in a previous case, but the exclusion may have been based on ¬B (they didn't properly test that sample) or ¬A (accreditation had lapsed at that time). If either antecedent condition was unmet previously, no contradiction with Rule 1 arises.

What the defence must prove: to exclude the evidence in this case, the defence must show either (i) A(LabX)=F — accreditation is currently invalid, or (ii) B(LabX)=F — proper testing procedures were not followed for this specific sample, so this x doesn't have both A and B, making Rule 1 inapplicable. Simply pointing to a prior exclusion is insufficient unless the defence can establish that it arose from ¬A or ¬B and that the same condition persists in this case.

Bar & LSAT Lens
NY Bar · Constitutional Law & LSAT Logical Reasoning

The quantifier-order problem (∀x∃y vs ∃y∀x) appears in contract drafting, constitutional law, and international treaty interpretation — the difference between "every state has some remedy" and "there is a single universal remedy for every state" has produced real legal disputes at the highest levels. A lawyer with quantifier-order fluency reads a contractual provision and immediately sees which structure is actually there, regardless of how ambiguous the English phrasing is.

LSAT Logical Reasoning's "Most Strongly Supported" and "Must Be True" questions are almost always quantifier-scope questions in disguise: if the stimulus contains a ∀, what can you infer about specific individuals (quantifier instantiation)? If it contains ∃, can you conclude a ∀? (Never — one example never establishes a universal.) These inferential limits are the predicate logic rules that drive the correct answer in roughly 30% of Logical Reasoning questions.
